# 技能栈与工具链

## 🐍 Python 生态

| 工具 | 场景 | 选型理由 |
|------|------|---------|
| **Playwright** | 爬虫API拦截 | Puppeteer被反爬后迁移，拦截XHR直接拿JSON |
| **pandas** | 数据清洗/分析 | 金融信号系统的数据管道核心 |
| **sqlite3** | 轻量数据存储 | 爬虫平台统一总表，拒绝过度工程化 |
| **NLTK** | 新闻文本预处理 | 金融新闻的分词、情绪词提取 |
| **requests** | 简单HTTP调用 | 比urllib更友好，保留urllib做底层备用 |

## ⚙️ TypeScript / Node.js

| 工具 | 场景 | 选型理由 |
|------|------|---------|
| **OpenClaw框架** | Agent调度/心跳/ cron | 原生支持多Agent协作、HEARTBEAT巡检 |

## 🗄️ 数据与存储

| 方案 | 场景 | 设计哲学 |
|------|------|---------|
| **SQLite单表** | 爬虫数据 | "一张总表就够了"，source字段区分来源 |

## 🛡️ 工程化实践

```
代码质量：      TypeScript严格模式 │ Result<T,E> │ 接口定义优先
错误处理：      3重重试 │ 指数退避 │ 自主修复记录
内存安全：      流式写入 │ BATCH_SIZE │ OOM防护
数据安全：      脱敏处理 │ 限流管控 │ 429退避
```
