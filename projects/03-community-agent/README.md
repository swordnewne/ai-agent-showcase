## 🎯 技术亮点

### 1. A/B 标题对比实验系统 v1.0

**问题**：编号档案型标题（`LEARNED-019: ...`）均评论仅 **0.4**，对比型标题（`从X到Y`）均评论 **6.0**，差15倍。

**方案**：同一素材生成3个标题变体，早晚分时段发布，3天后自动复盘。

```python
def generate_title_variants(base_title, content):
    """生成3种标题变体"""
    variants = [
        ("pain_action", f"从'{痛点}'到{结果}：{方案}"),      # 早间 8:00
        ("number_shock", f"{数字}{单位}的代价：{教训}"),     # 晚间 20:00
        ("counter_intuitive", f"为什么{常识}其实是{反常识}"), # 备用
    ]
    return variants
```

**评分公式**：`综合得分 = 点赞×1 + 评论×2 + 浏览×0.1`

### 2. 标题清洗策略

```python
def clean_title(title):
    """去除降低点击率的噪音"""
    # 去除 LEARNED-XXX 编号前缀
    title = re.sub(r'^LEARNED-\d+[:：]\s*', '', title)
    # 去除日期前缀
    title = re.sub(r'^\d{4}-\d{2}-\d{2}\s+', '', title)
    # 去除通用修饰词
    title = re.sub(r'^(踩坑档案|事故档案|补充)[：:]\s*', '', title)
    return title
```

**清洗效果**：
| 清洗前 | 清洗后 |
|--------|--------|
| `LEARNED-019: 创业方向验证...` | `创业方向验证结构化框架` |
| `踩坑档案：定时任务优化...` | `定时任务优化：从 16 次无效采集到精准触发` |

### 3. 动态素材生成（告别硬编码）

旧方案：4个硬编码模板，发完就卡住。  
新方案：从 `.learnings/` 踩坑档案动态提取"问题+方案+结果"：

```python
def collect_materials():

        f'{LEARNING_DIR}/incidents/**/*.md',   # 踩坑档案

        'skills/*/SKILL.md',                    # 技能文档
        'ai-intelligence-radar/reports/*.md',   # 情报雷达
    ]
    # 提取结构化内容：问题/方案/结果/量化数据
    return [parse_material(f) for f in scan_sources(sources)]
```
