# 反模式与修复

> 知道什么不对，比知道什么对更重要。这些是我们踩过的坑。

---

## 反模式1：虚假层级（False Hierarchy）

### 症状

平级模块画成上下级关系，读者误以为存在触发或包含关系。

### 示例

```
❌ 错误：
    [系统告警]
       ↓
    [社区学习]
    
读者会以为：系统告警触发社区学习
实际上：两者是平级处理器，互不隶属
```

### 修复

```
✅ 正确：
    [系统告警]    [社区学习]
         ↓            ↓
      [共享数据层]
```

### 检查方法

对每一对上下相邻的节点问："A是B的父级吗？A会触发B吗？"

---

## 反模式2：意大利面（Spaghetti）

### 症状

箭头交叉过多，N×M条连接线。

### 示例

```
❌ 错误（4×4=16条箭头）：
    [A1] → [B1]  [A1] → [B2]  [A1] → [B3]  [A1] → [B4]
    [A2] → [B1]  [A2] → [B2]  ...
```

### 修复

```
✅ 正确（用总线，8条箭头）：
    [A1] ──┐
    [A2] ──┼──→ [总线] ──→ [B1] [B2] [B3] [B4]
    [A3] ──┘
```

## 反模式3：信息过载（Information Overload）

### 症状

单图超过15个节点，或单层超过7个节点。

### 修复

**方法A：抽象**
```
原始：15个具体服务
抽象：3个服务组，每个组标注"（含5个服务）"
```

**方法B：拆分**
```
图1：系统 Overview（7个节点）
图2：服务组A详情（展开）
图3：服务组B详情（展开）
```

## 反模式4：对比度灾难（Contrast Failure）

### 症状

浅色背景+浅色文字，或鲜艳颜色+默认黑字。

### 示例

```
❌ 错误：
    style C fill:#f9f,stroke:#333
    #f9f(浅粉)背景 + GitHub默认白字 = 完全看不见
```

### 修复

```
✅ 正确：
    style C fill:#a371f7,stroke:#fff,color:#000
    #a371f7(紫)背景 + 黑字 = 对比度4.8:1
```

---

## 反模式5：缺失上下文（Missing Context）

### 症状

图只有节点和箭头，没有：
- 图标题（这张图讲什么）
- 图例（颜色/形状的含义）
- 环境说明（这是生产架构还是开发架构）

### 修复

每张图必须包含：
```
标题：XX系统架构 Overview
图例：蓝色=事件源，绿色=业务模块，紫色=数据服务
环境：生产环境（3节点集群）
版本：v2.1（2026-06-26）
```

---

## 反模式6：单向思维（One-Way Thinking）

### 症状

只画正常流程，不画：
- 错误路径
- 降级策略
- 超时处理

### 修复

```
✅ 正确：
    [请求] → [处理] → [成功]
              ↓
           [失败] → [重试] → [降级]
```

---

## 反模式7：过度抽象（Over-Abstraction）

### 症状

图太抽象，失去了具体信息。

```
❌ 错误：
    [前端] → [后端] → [数据库]
    （这图对任何系统都适用，信息量=0）
```

### 修复

```
✅ 正确：
    [React前端] → [FastAPI网关] → [PostgreSQL主从]
    （具体技术栈，信息量>0）
```

## 诊断清单

| 反模式 | 检查方法 | 严重程度 |
|--------|---------|---------|
| 虚假层级 | 每对相邻节点问因果关系 | 🔴 致命 |
| 意大利面 | 数箭头数，N×M>20 | 🟠 严重 |
| 信息过载 | 数节点数>15 | 🟠 严重 |
| 对比度灾难 | GitHub dark mode预览 | 🔴 致命 |
| 缺失上下文 | 检查标题/图例/环境 | 🟡 中等 |
| 单向思维 | 检查是否有错误路径 | 🟡 中等 |
| 过度抽象 | 图是否适用于其他系统 | 🟡 中等 |
